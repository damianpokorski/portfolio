Automated deployment

This is an interesting one. I am tasked with the continuing development of the already established set of scripts which automate the deployment of the product to the customers. The technologies utilised here are : a large pile of Git, Jenkins, powershell scripts, dotnet core, unit testing, Azure Cloud services, OctopusDeploy. Our deployment process utilises a fairly complex multistage path:

  1. 1. Cleaning the solution
  2. 2. Restoring packages
  3. 3. Building the solution
  4. 4. Executing unit tests
  5. 5. Packaging the solution
  1. 6. Publishing generated package to OctopusDeploy
  2. 7. Creating a network of virtual machines within Azure
  3. 8. Deploying to the Azure through OctopusDeploy
  4. 9. Executing additional simulation tests
  1. 10. Evaluating the tests results
  2. 11. Automated benchmarking
  3. 12. Scaffolding simulation databases
  4. 13. Comparing results
Over all I have developed a taste for DevOps with this being my main responsibility. I have developed a fair understanding and abilitity to automate most of the tasks (I even managed to automate Tableau workbook publishing, which was fairly tricky). I guess I have a knack for automating things,
